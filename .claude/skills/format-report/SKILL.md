# ai_news レポート整形

## 対象ファイル
- `ai_news/report/*.md`。ユーザーが対象を指定しなければ、現在IDEで開いているファイル、それも無ければ `report/` 配下で直近更新されたファイルを対象にする。

## 前提
このフォルダのレポートは、生成AIが出力した長文がほぼ改行なしの1〜数行の塊としてそのまま保存されていることが多い。見出し・箇条書き・表が、番号や接頭ラベルだけを残してすべて地の文に溶け込んでいる。このスキルの役割は、失われた構造を文面から復元して読みやすいMarkdownに組み直すことであり、要約・加筆・削除は行わない。

## 手順

1. 対象ファイルを全文読む。
2. 文中に埋もれている構造の手がかりを見つける。
   - `1.`, `2.1`, `3.3.1` のような番号付き見出しラベル → 番号の深さに応じて `#`（タイトル）/`##`（大項目）/`###`（中項目）に変換する。
   - 「ラベル: 説明文」の形で並んでいる短い項目（例: 「技術的特異性: Devinは…」「対策: …」）→ `- **ラベル**: 説明文` の箇条書きに変換する。
   - 「固有名詞＋コロン風の見出し句」のあとに説明段落が続く構造（例: 「Devin (Cognition AI): 「不休のチームメイト」としての自律性」）→ 見出し句を `**太字**` の小見出し行にし、続く説明は通常の段落にする。
   - 「表N: タイトル」に続けてヘッダー列名と各セルの値がすべて連結されているブロック → 列名・行ラベル・セル値の繰り返しパターンを読み取り、Markdownテーブル（`| … | … |` とヘッダー区切り行）に再構築する。セルの境界が曖昧な場合は、文意が通る自然な区切りで分割してよい（このときも情報の追加・削除はしない）。
3. タイトル直下に副題があれば `**副題**` として独立させる。
4. 「。」の前の余分な半角スペース（引用符号が取り除かれた痕跡）など、変換に伴う軽微な表記ゆれは整えてよい。ただし文章の意味・語順・データを一切変更しない。
5. セクションの区切りには `---` を使い、視覚的な間隔を作る。
6. 整形後の内容を同じファイルパスに上書き保存する。
7. 編集後、見出し・表を何個再構築したか程度を1〜2文でユーザーに要約報告する（ファイル全文を貼り直さない）。
